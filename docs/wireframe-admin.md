# Wireframe: Админ-панель — Калды.ру

> Документ описывает структуру и содержимое wireframe админ-панели.
> Используется как основа для разработки.
> Только десктоп. Доступна по `/admin`.
> Авторизация: статичный логин + пароль, JWT (см. `openapi.yaml` → `POST /auth/login`).

---

## Общая структура (все страницы кроме логина)

**Раскладка:** фиксированный сайдбар слева + контентная область справа.

```
┌─────────────────────────────────────────────────────┐
│  Navbar (топбар)                                    │
├──────────────┬──────────────────────────────────────┤
│              │                                      │
│   Сайдбар    │        Контентная область            │
│   (~220px)   │        (остаток ширины)              │
│              │                                      │
└──────────────┴──────────────────────────────────────┘
```

---

## Топбар

**Высота:** 52px  
**Фон:** `#1A2C35` (bg-dark)

**Содержимое:**
- Слева: логотип «Калды.ру» + подпись «Панель управления» (muted)
- Справа: имя пользователя «Администратор» + кнопка «Выйти» (ghost, белый текст)

---

## Сайдбар

**Ширина:** 220px фиксированная  
**Фон:** `#F5F9FB`  
**Граница:** `1px solid #C8DDE5` справа

**Навигация (сверху вниз):**

| Пункт | Иконка | URL |
|-------|--------|-----|
| Номера | 🏠 | `/admin/rooms` |
| Услуги | ✦ | `/admin/services` |
| Галерея | 🖼 | `/admin/gallery` |

**Стили пунктов:**
- Обычный: текст `#4A6572`, фон прозрачный
- Активный: фон `#1A6B8A15`, текст `#1A6B8A`, левая рамка `3px solid #1A6B8A`
- Hover: фон `#EAF4F8`

**Внизу сайдбара:**
- Версия: «v1.0» (muted, 11px)

## Экран 1: Логин `/admin/login`

**Раскладка:** центрированная карточка на фоне `#F5F9FB`

**Карточка (400px ширина):**
- Логотип / название «Калды.ру» сверху по центру
- Подзаголовок: «Панель управления»
- Разделитель
- Поле «Логин» (инпут, placeholder «admin»)
- Поле «Пароль» (инпут type=password)
- Кнопка «Войти» (primary `#E8A838`, 100% ширина)
- Текст ошибки под кнопкой (красный, скрыт по умолчанию): «Неверный логин или пароль»

**Поведение:**
- При успешном логине → редирект на `/admin/rooms`
- JWT сохраняется в `httpOnly cookie` или `localStorage`
- Защита роутов: если токен отсутствует/истёк → редирект на `/admin/login`

## Экран 3: Форма номера `/admin/rooms/new` и `/admin/rooms/:id/edit`

**Раскладка:** одна колонка, max-width 720px, по центру контентной области

**Заголовок:** «Новый номер» / «Редактировать: Коттедж №7»  
**Хлебные крошки:** `Номера › Новый номер`

### Секция «Основная информация»

| Поле | Тип | Обязательное |
|------|-----|--------------|
| Название | text input | ✓ |
| Slug | text input (авто-генерируется из названия, редактируемый) | ✓ |
| Тип жилья | select: Коттедж / Корпус / Домик | ✓ |
| Вместимость (основная) | number input | ✓ |
| Вместимость с допместами | number input | |
| Описание | textarea (4 строки) | |

### Секция «Цены»

| Поле | Тип |
|------|-----|
| Цена (будни), ₽/сут | number input |
| Цена (выходные/праздники), ₽/сут | number input |
| Цена допместа, ₽/сут | number input |

### Секция «Удобства и условия»

| Поле | Тип |
|------|-----|
| Постельное бельё | checkbox |
| Популярный (featured) | checkbox |
| Удобства | tag-input (вводишь строку → Enter → добавляется тег) |
| Заезд с | time input (default «14:00») |
| Выезд до | time input (default «12:00») |
| Условия с животными | text input |
| Примечание об оплате | text input |

### Секция «Фотографии»

**Компонент загрузки фото:**
- Зона drag-and-drop («Перетащите фото или нажмите для выбора»)
- Поддержка множественной загрузки
- После загрузки: сетка миниатюр 3 колонки

**Каждая миниатюра:**
- Фото 120×90px
- Поле alt-текста под фото (inline редактирование)
- Иконка «⋮⋮» слева — drag handle для изменения порядка
- Бейдж «Обложка» на первом фото (order = 0)
- Кнопка «✕» — удалить фото
- Чекбоксы категорий галереи: `Природа · Пляж · Территория · Номера · Мероприятия`

**Поведение:**
- Порядок меняется drag-and-drop → `PUT /admin/gallery/reorder`
- Первое фото автоматически становится обложкой

### Кнопки формы (внизу, sticky)

- «Сохранить» — primary `#E8A838`
- «Отмена» — outline `#1A6B8A` → возврат в список

**Валидация:**
- Ошибки показываются inline под полем
- Кнопка «Сохранить» неактивна если не заполнены обязательные поля

## Экран 6: Галерея `/admin/gallery`

### Шапка страницы
- Заголовок H1: «Фотогалерея»
- Кнопка «+ Загрузить фото» справа (primary `#E8A838`)

### Фильтр по категориям
Чипы: «Все · Природа и озеро · Пляж · Территория · Номера · Мероприятия»  
Справа: счётчик «Фото: N»

### Сетка фото

**Тип:** 5 колонок, зазор 8px  
**Каждая ячейка:**
- Фото 100% ширина, aspect-ratio 4/3, `object-fit: cover`
- При hover — тёмный оверлей с действиями:
  - Иконка «⋮⋮» (drag handle) — для изменения порядка
  - Иконка «🏷» — редактировать категории
  - Иконка «✕» — удалить

**Drag-and-drop:**
- Фото перетаскиваются внутри категории
- При отпускании → `PUT /admin/gallery/reorder`
- Визуальный placeholder на месте перетаскиваемого фото

**Важно:** порядок drag-and-drop работает только внутри выбранной категории. При фильтре «Все» — drag-and-drop отключён, показывается подсказка «Выберите категорию для изменения порядка».

### Модальное окно «Редактировать категории»

Открывается по иконке «🏷» на фото.

**Содержимое:**
- Превью фото (120px)
- Поле alt-текста
- Чекбоксы категорий:
  - ☑ Природа и озеро
  - ☐ Пляж
  - ☑ Территория
  - ☐ Номера
  - ☐ Мероприятия
- Кнопки «Сохранить» / «Отмена»

### Модальное окно «Загрузить фото»

Открывается по кнопке «+ Загрузить фото».

**Содержимое:**
- Зона drag-and-drop (множественная загрузка)
- После выбора файлов — список превью с полями:
  - Alt-текст (инпут)
  - Категории (чекбоксы, обязательно выбрать хотя бы одну)
- Кнопка «Загрузить N фото» / «Отмена»

## Общие UX-решения

### Уведомления (toast)
- Успех: зелёный тост справа сверху — «Номер сохранён», «Фото удалено» и т.д.
- Ошибка: красный тост — «Ошибка сервера. Попробуйте ещё раз.»
- Длительность: 3 секунды, закрывается крестиком

### Состояния загрузки
- Кнопка «Сохранить» во время запроса: спиннер + текст «Сохранение...», неактивна
- Таблица при загрузке: skeleton-строки

### Защита роутов
- Middleware Next.js проверяет JWT на каждый запрос к `/admin/*`
- Истёкший токен → редирект на `/admin/login` с параметром `?redirect=/admin/rooms`
- После логина → редирект обратно на исходный URL
