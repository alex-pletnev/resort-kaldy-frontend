# Бриф: Сайт-визитка базы отдыха

## О проекте

Сайт-визитка для базы отдыха с каталогом номеров и услуг.

**Название:** Калды.ру  
**Целевая аудитория:** семейные пары 40–70 лет, часто с детьми и внуками.  
**Природа:** лес + озеро с пляжем (лежаки). Главная фишка — отдых у воды.  
**Главная цель:** привлечь гостей — рассказать о базе, показать номера и услуги, вызвать желание приехать.

**Вторичные цели:**
- Показать все номера с описанием, фото и ценами
- Показать все услуги с описанием и ценами
- Дать официальную информацию (документы, правила проживания)

## Страницы сайта

### Обязательные

| Страница | Содержимое |
|---|---|
| Главная | Hero с фото природы, название базы, УТП, краткое "О нас", превью номеров, превью услуг, контакты, FAQ секция, как добраться секция |
| Каталог номеров | Все номера: фото, краткое описание, цена. Фильтрация и сортировка. Переход в карточку номера |

| Каталог услуг | Все услуги: фото, краткое описание, цена. Фильтрация. Переход в карточку услуги |
| Карточка услуги | Полное описание, фото, цена |
| Документы | Официальная информация, правила проживания, правила поведения — статичные PDF/текст |
| Фотогалерея | Фотографии базы, природы, номеров, мероприятий |
| Как добраться | Карта, маршруты, адрес, схема проезда |

### FAQ
Секция на главной странице (не отдельная страница) — так меньше кликов для аудитории 40–70 лет.

### Админ-панель (под авторизацией)
- Управление номерами: создание, редактирование, удаление, загрузка фото
- Управление услугами: аналогично
- Доступна по отдельному URL, например `/admin`

## Функциональные требования

### Бронирование
Сейчас: кнопка "Забронировать" → переход к контактам (телефон / мессенджер).  
В будущем: онлайн-бронирование с выбором дат и оплатой (заложить в архитектуру).

### Контент
- Номера и услуги: динамически через REST API
- Всё остальное: статика (Next.js SSG/SSR)
- Язык: только русский

### Фильтрация и сортировка (каталоги)
Номера — фильтры: вместимость, тип, цена. Сортировка: по цене, по популярности.  
Услуги — фильтры: категория. Сортировка: по цене.

### Логика «Похожие номера» и «Похожие услуги»
**Номера** — приоритет подбора:
1. По вместимости (совпадение ±1)
2. По цене (ближайшие по priceWeekday)

**Услуги** — подбор по категории.

## Технический стек

| Слой | Технология |
|---|---|
| Фронтенд | Next.js 14+ (App Router) + Tailwind CSS |
| Бэкенд | Java / Kotlin — RESTful API |
| Деплой фронта | Vercel |
| Авторизация (админка) | JWT через API |
| Хранилище фото и файлов | Яндекс Object Storage |

**Обоснование Next.js:** SSR/SSG для SEO, роутинг из коробки, хорошо работает с REST API, отлично поддерживается Claude Code.

**Схема работы с фото:**
```
Админка → загрузка на бэкенд → бэкенд делает ресайз → сохраняет в Яндекс Object Storage → URL в БД → API отдаёт URL → фронт показывает фото
```

### API
Контракт между фронтендом и бэкендом описан в файле `openapi.yaml` (Project Knowledge).  
Спецификация покрывает:
- Публичные эндпоинты: номера, услуги, похожие, featured
- Админские эндпоинты (JWT): CRUD номеров и услуг, загрузка фото
- Авторизация: `POST /auth/login`, `POST /auth/refresh`

## Фотографии

### Хранилище
Все фото хранятся в Яндекс Object Storage, бакет **`kaldy-media`**.

**Базовый URL:** `https://storage.yandexcloud.net/kaldy-media/`

**Пример ссылки на фото:**
```
https://storage.yandexcloud.net/kaldy-media/rooms-building-2-room-1-photo-1.jpg
```

### Правила нейминга файлов

Все файлы лежат в корне бакета (без подпапок). Категория и принадлежность зашиты в имя файла.

**Схема:** `{категория}-{slug}-{photo-N}.jpg`

| Категория | Схема | Пример |
|---|---|---|
| Номер (с подномерами) | `rooms-{cottage-slug}-{room-slug}-photo-{N}.jpg` | `rooms-cottage-1-room-2-photo-3.jpg` |
| Номер (без подномеров) | `rooms-{cottage-slug}-photo-{N}.jpg` | `rooms-gorodok-photo-1.jpg` |
| Территория | `territory-photo-{N}.jpg` | `territory-photo-5.jpg` |
| Услуги | `services-photo-{N}.jpg` | `services-photo-2.jpg` |

### Slug номеров

| Папка оригинал | Slug |
|---|---|
| 1 дом готов | `cottage-1` |
| 14 дом готов | `cottage-14` |
| 8 коттедж готов | `cottage-8` |
| Корпус 2 | `building-2` |
| корпус 9 готов | `building-9` |
| городок | `gorodok` |
| рыбацкие / рыбацкий 2 | `fishing-houses` / `fishing-2` |
| цветные / зеленый | `colored-houses` / `green` |
| цветные / коричневый | `colored-houses` / `brown` |

### Подготовка фото
Фото конвертируются из `.heic` в `.jpg` скриптом `prepare_photos.py`.  
Скрипт берёт оригиналы из `/Users/aleksandrpletnev/Downloads/База все фото`, конвертирует и переименовывает по схеме выше, складывает все файлы в плоскую папку `~/Desktop/kaldy-upload/` готовую к загрузке в Object Storage.
